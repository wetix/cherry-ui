# Docs

## Folder structure

Stories for components should be in the stories folder, with a subfolder for the component.
Inside the subfolder should have a stories file, for testing the component and docs file for documentation.
```
/stories
  /Component
    Component.docs.mdx
    Component.stories.svelte
```

### Component.stories.svelte

```
<script>
```

# Test cases

Test cases are created to make sure the components work properly on release.

`@testing-library/svelte` and `jest` is used for testing the individual components.

For creating tests for components, please refer to below:

## Boilerplates

### Test boilerplate
```
import { render, fireEvent } from '@testing-library/svelte';
import Component from "../src/Component.svelte";
import SlotTest from "../../../test/slot/SlotTest.svelte";

describe("Component test", () => {
  const props = {

  };

  it("should render properly", () => {
    const result = render(Component, { props });
    expect(() => result).not.toThrow();
  });

  it("should render with slots", () => {
    const result = render(SlotTest, { props: { component: Component, props } });
    expect(() => result).not.toThrow();
  });

  // getElementById, getElementsByClassName, getElementsByTagName may be used instead of querySelector:
  it("should have correct props", () => {
    const { container } = render(Component, { props });
    const component = container.querySelector("") as HTMLElement;
  });

  it("fireEvent", async () => {
    const { container } = render(Component, { props });
    const component = container.querySelector("") as HTMLElement;
  });
});
```
